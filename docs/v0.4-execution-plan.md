# v0.4 Execution Plan — Autonomous Finance + M2M Agent Commerce on Base

- **Status:** Active
- **Date:** 2026-05-26
- **Design:** [RFC 0001](rfcs/0001-autonomous-finance-m2m-commerce-base.md)
- **Decision:** Extend `brain-core` in place (no rewrite, no new repo). New
  surfaces ship as additive packages/contracts on the existing spine.

This is the plan of record for delivering RFC 0001. It translates the RFC's
design into phased, individually-shippable PRs against `main`.

## Operating rules (how we execute)

1. **Spec-first → shadow-first → promote.** Update the source-of-truth spec
   (OpenAPI / Standards / Architecture) → land code **shadowed/flagged** →
   exercise → promote via `scripts/check-promotion-readiness.mjs`.
2. **Enforcement before features.** Land the CI guards
   (`check-no-onchain-pii`, doc-drift) and generate-from-source tooling early so
   new work cannot reintroduce drift or put data on-chain.
3. **Incremental merge to `main` behind the shadow gate — no long-lived
   branches.** Money-movers are shadow-by-default, so new code is safe to sit on
   `main` dark. Small PRs, merged continuously; flip behavior via `LIVE_AGENTS`.
4. **`main` stays green every step.** Small, reviewed, `ai-assisted` PRs (TDD).
5. **Docs move with code, page-by-page, status-tagged.** Never batch docs at the
   end — that is how the GitBook drift happened.
6. **No PII/financial detail on-chain — ever.** Commitments only (RFC §3).
7. **No money contract to mainnet without an external audit** (RFC §9).

## Documentation + GitBook strategy

1. Reconcile existing drift first (PR2) so GitBook is honest before adding to it.
2. Land the doc-drift CI guard + generate-from-source for API/errors/scopes (PR3).
3. Tag every relevant page **Shipped / In-Progress / Planned** (RFC §12 matrix).
4. Add new pages as **Planned**; flip to **Shipped** as each feature lands;
   promote committed pieces into the spec hierarchy.
5. GitBook auto-syncs from `main` on merge — docs ride the normal PR flow.

## Out of scope (RFC §13)

Custody; PII/financial detail on-chain; replacing the deterministic gate with
reputation/LLM judgment; a parallel un-gated payment path; upgradeable contracts
by default.
